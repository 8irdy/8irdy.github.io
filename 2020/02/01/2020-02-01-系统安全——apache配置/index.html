一、配置虚拟主机
1、配置基于域名的虚拟主机
举例：DNS服务器为192.168.10.1，web服务器为192.168.10.1
所有人通过域名：www.zlt.com访问lzt公司的网站。
所有人通过域名：www.slxy.com访问shuili公司的网站。
所有人通过域名：www.crt.com访问crt公司的网站。

第一步：搭建DNS服务器
分别建立zlt.com shuli.com crt.com三个域
主机记录分别为：www.zlt.com 192.168.10.1
www.shuili.com 192.168.10.1
www.crt.com 192.168.10.1
第二步：创建站点主目录及首页文件

#mkdir /var/www/zlt

#echo ‘welcome to www.zlt.com!!’ > /var/www/zlt/index.html

#mkdir /var/www/shuili

#echo ‘welcome to www.shuili.com!!’ > /var/www/shuili/index.html

#mkdir /var/www/crt

#echo ‘welcome to www.crt.com!!’ > /var/www/crt/index.html

第三步：编辑配置文件
在httpd.conf文件的“Virtual Hosts”部分增加如下内容。
NamevirtualHost 192.168.10.10:80
<virtualHost 192.168.10.1:80>
DocumentRoot /var/www/lsy
ServerName www.zlt.com
DirectoryIndex index.html

<virtualHost 192.168.10.1:80>
DocumentRoot /var/www/shuili
ServerName www.shuili.com

<virtualHost 192.168.10.1>
DocumentRoot /var/www/crt
ServerName www.crt.com

第四步：测试验证
在浏览器中分别输入以下网址进行测试：
www.lsy.com www.shuili.com www.crt.com

2、配置虚拟目录
虚拟目录是主网站的下一级子目录，要依附于主网站，但它的物理位置并不在主目录下。
第一步：创建虚拟目录

# mkdir /movie

echo ‘virtual directory’> /movie/index.html

第二步：修改httpd.conf主配置文件
<virtualHost 192.168.10.1>
DocumentRoot /var/www/zlt
ServerName www.zlt.com
Alias /movie “/movie” （/movie 网站路径 “/movie” 真实路径）
</virtualHost 192.168.10.1>

第三步：访问测试
在浏览器中输入www.zlt.com/movie进行测试，查看是否成功。

四、设置目录权限
（1）基于客户端地址的访问控制
目录的访问控制可以通过Order、Allow、Deny等语句来实现，Order语句用于定义默认的访问权限，Allow语句用于定义允许访问目录的主机列表，Deny语句用于定义拒绝访问目录的主机列表。
Order语句的用法主要有两种：
Order allow,deny 默认拒绝所有客户端访问
Order deny,allow 默认允许所有客户端访问
Allow和Deny语句的用法：
Allow from 192.168.80.0/24 允许192.168.80.0网段内的客户端访问
Deny from 192.168.80.101 拒绝IP为192.168.80.101的客户端访问

举例说明：
NameVirtualHost 192.168.0.1:80
<VirtualHost *:80>
DocumentRoot /var/www/slxy/
ServerName www.cqjd.com
DirectoryIndex index.html
Alias /virtual “/virtual”
<Directory “/var/www/slxy”>
order allow,deny
allow from 192.168.80.0

NameVirtualHost 192.168.0.1:80
          <VirtualHost *:80>
               DocumentRoot /var/www/cqjd/
               ServerName www.cqjd.com
               DirectoryIndex index.html
               Alias /virtual "/virtual"
               <Directory "/virtual">
                    order deny,allow
                    deny from 192.168.0.10                   
               </Directory>
          </VirtualHost>

（2）基于用户的访问控制
基于用户的访问控制也是只能针对服务器上的某个目录进行设置，设置内容必须包含在<Directory 目录> …… 的区域中。
AuthName 定义受保护的领域名称，客户端访问时在弹出的认证登录对话框中将显
示该名称
AuthUserFile 设置用于保存用户帐号、密码的认证文件路径
AuthType 配置认证类型（基本认证：Basic 摘要认证：Digest）
require valid-user 授权给认证文件中的所有有效用户
如果写成 require user [用户名]
表示只允许该用户访问
举例说明：
第一步：修改配置文件httpd.conf，在www.zlt.com的虚拟主机部分中添加如下的内容：
<virtualHost 192.168.10.1>
DocumentRoot /var/www/zlt
ServerName www.zlt.com
Alias /movie “/movie”
<Directory “/movie”>
AuthName “movie”
AuthType Basic
AuthUserFile /etc/httpd/conf/.htpasswd
require valid-user

</virtualHost 192.168.10.1>
第二步：创建.htpasswd文件并向其中添加用户
对认证用户进行管理需要用到htpasswd命令，执行下面的命令来创建.htpasswd文件并向其中添加harry用户。
[root@localhost ~]# htpasswd -cm /etc/httpd/conf/.htpasswd harry
htpasswd命令：
-c 创建认证文件
-m MDB5加密
-D 删除一个用户
第三步：访问测试
最后在客户端访问测试，在访问“www.example.com/movie”时提示要输入用
户名和密码。成功通过验证后，可以顺利访问到该页面。
