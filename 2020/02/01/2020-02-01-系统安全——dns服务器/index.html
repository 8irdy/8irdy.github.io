DNS服务器

项目简介：

某公司内部网络较多，为了使员工能方便记忆网络地址，将配置局域网DNS服务器，该服务器能够正反向解析下面的域名与IP地址的对应关系。如果当前DNS服务器不能解析，则将解析请求上传到一个目前比较权威的DNS服务器219.216.129.5解析。

www.excesoft.com 192.168.137.42

mail.excesoft.com 192.168.137.16

exam.excesoft.com 192.168.137.5

attendance.excesoft.com 192.168.137.6

dns.excesoft.com 192.168.137.22

目前，操作系统的IP是192.168.137.22。

项目分析：

DNS是Server/Client工作模式，要完成这个项目管理员首先需要准备一台Centos7的服务器和一台windows客户端，并在服务器上安装DNS，启动named服务并设置开机自启。然后根据项目需求对DNS服务器进行配置。最后进行DNS服务器的测试。

项目实施：

一、网络环境的准备

1、CentOS7服务器与windows XP客户端各一台。

2、设置网络环境与IP地址。

3、测试网络的连通性。

二、服务器端的配置

1、安装DNS服务器

（1）安装bind程序包：

#yum install bind

（2）启动服务

# systemctl start named.service

（3）设置自启动状态

# systemctl enable named.service

2、主配置文件配置：#vim /etc/named.conf

在文件末尾添加加载辅助区域文件的内容：include “/etc/named.rfc1912.zones”

3、配置named.rfc1912.zones文件：#vim named.rfc1912.zones

4、配置区域配置文件

（1）正向解析配置

为excesoft.com域名设计正向解析，将/var/named/named.empty复制重命名为excesoft.com.zone，

修改文件属组为named，并对其内容进行修改。

（2）反向解析配置

为excesoft.com域名设计反向解析，将/var/named/named.empty复制重命名为137.168.192.zone，修

改文件属组为named，并对其内容进行修改。

5、启动DNS服务器：#systemctl start named.service

named-checkconf 检查dns配置文件的方法是否正确

二、客户端的配置

DNS服务器IP是192.168.137.22，客户端可通过DHCP来自动获取DNS地址，也可手动设置DNS地址。

1.修改配置文件：#vim /etc/resolv.conf。

写入以下内容：

​ excesoft.

nameserver 192.168.137.22

3.测试DNS服务器

在客户端计算机上使用命令 nslookup进行测试。
